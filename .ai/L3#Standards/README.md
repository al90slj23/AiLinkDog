# L3#Standards

`L3#Standards` 是长期规范层，用于保存当前项目稳定、可复用、可长期维护的详细规则。

## 本层职责

- 承载当前仓库的详细开发规则
- 将根级约束整理成长期可引用文档
- 为未来 AI 和开发者提供深入规则正文

## 当前结构

- `standards/`：主规范目录

`standards/` 下的正文文件统一使用单词链命名，用链式语义表达规范主题，而不是使用并列短语或复合短语兜底。

## 维护原则

- 当前仓库真实状态优先
- 当前代码与根级规范优先
- 当 L1/L2 中的概览发生变化，应同步检查 L3 是否需要更新
- `standards/` 正文文件统一使用单词链命名
- 命名与拆文都应以正文真实主主题为准
- 新增规范时优先增层或拆文，不使用复合短语兜底
- README 负责导航与维护约束，详细主题入口以 `toc.md` 与 `standards/README.md` 为准
