# 后端规范：JSON 与 DTO

## JSON 规则

业务代码中的 JSON 编解码必须使用 `common/json.go` 提供的包装函数，不应直接调用标准库 `encoding/json` 的 Marshal / Unmarshal。

## DTO 规则

对于客户端 JSON 输入后又要转发到上游 provider 的请求结构体，所有可选标量字段必须使用指针类型，以保留显式零值。

推荐语义：

- 字段缺失：`nil`，序列化时省略
- 显式传入 `0` / `0.0` / `false`：非 `nil` 指针，序列化时必须保留

## 关键原则

- 缺省值与显式零值必须语义分离
- DTO 的设计应服务于 relay / convert 的真实传输语义
- 对 `json.RawMessage` 等类型可以继续引用标准库类型定义，但实际编解码调用仍应走 `common.*`
