# 后端规范：DTO

## 适用范围

适用于客户端 JSON 输入后还要继续转发到上游 provider 的请求结构体，尤其是 `relay` / `convert` 路径下的 DTO 设计。

## 核心规则

- 对于客户端 JSON 输入后又要转发到上游 provider 的请求结构体，所有可选标量字段必须使用指针类型。
- 推荐语义如下：
  - 字段缺失：`nil`，序列化时省略。
  - 显式传入 `0` / `0.0` / `false`：非 `nil` 指针，序列化时必须保留。
- DTO 的设计应服务于 relay / convert 的真实传输语义。

## 注意事项

- 缺省值与显式零值必须语义分离，避免转发时无意丢失上游参数。
- 当 DTO 涉及 JSON 编解码实现时，实际调用仍应遵循 `02.backend-01.json.md` 中的 `common/json.go` 约束。
