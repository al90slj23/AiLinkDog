# 后端规范：JSON

## 适用范围

适用于后端业务代码中的 JSON 编解码实现，以及需要在项目内统一 JSON 调用入口的场景。

## 核心规则

- 业务代码中的 JSON 编解码必须使用 `common/json.go` 提供的包装函数。
- 不应直接调用标准库 `encoding/json` 的 `Marshal` / `Unmarshal`。
- 对 `json.RawMessage` 等类型可以继续引用标准库类型定义，但实际编解码调用仍应走 `common.*`。

## 注意事项

- 统一走 `common.*` 是为了保持项目内 JSON 行为一致，并为后续替换底层实现保留空间。
- 当规则涉及 DTO 的显式零值语义时，应同时参考 `02.backend-02.dto.md`。
