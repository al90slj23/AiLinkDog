# AI 记忆体系使用方式

## AI 使用顺序

1. 先读 `L1#Overview/guide.md`
2. 再看 `L2#Index/toc.md`
3. 根据主题进入 `L3#Standards/standards/`
4. 需要历史时看 `L4#Changelog/`
5. 需要建系统图时看 `L5#Knowledge/system-map.md`

## 何时更新

- 技术栈变化时更新 L1 / L2 / L3
- 核心规则变化时更新 L1 / L2 / L3，并在 L4 记日志
- 新的关系认知出现时更新 L5

## 当前附加约束

- `gogogo.sh 7` 产生的持续追加型“工作日志”默认归入 `L4#Changelog/`
- 这类工作日志应使用 `YYYY-MM-DD-HHMM-主题.md` 的时间戳命名，不使用 `recent-*`、`latest-*` 这类滚动命名
- `L5` 用于系统关系、结构认知与长期知识，不承载这类持续追加型工作日志

## 维护原则

- 只写当前项目真实情况
- 不继承冲突旧规则
- 默认中文
