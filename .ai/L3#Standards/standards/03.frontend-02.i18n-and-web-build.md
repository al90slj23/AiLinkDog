# 前端规范：i18n 与构建

## i18n

前端使用 `i18next`，翻译文件位于 `web/src/i18n/locales/`。

后端也有自己的 i18n 体系，二者不可混淆。

当前前端语言包括：`zh`、`en`、`fr`、`ru`、`ja`、`vi`。

翻译 key 使用中文源字符串，修改文案时需要注意这一点。

## 术语一致性

当前仓库存在 `docs/translation-glossary.md` 术语表，说明项目已经开始维护翻译术语一致性。

涉及多语言文案修改时，应优先保持以下概念语义稳定：

- 倍率 / Ratio
- 令牌 / Token
- 渠道 / Channel
- 分组 / Group
- 额度 / Quota
- 提示 / Prompt
- 补全 / Completion
- 充值 / Top Up
- 兑换码 / Redemption Code

如果未来 `.ai` 扩充翻译规范，应优先从这份术语表继续整理，而不是临时逐词翻译。

## 构建

- 前端构建使用 `bun run build`
- 构建输出位于 `web/dist`
- 后端 `main.go` 对 `web/dist` 有 embed 依赖
- `web/dist/index.html` 在当前仓库里还承担后端 embed 占位职责

## 初始化入口

从 AiLinkDog 开始，前端不再把多步初始化向导作为主路径保留。

数据库和运行模式不再通过引导式 Setup Wizard 向用户传达“多种正式选择”。

## 影响

前端构建产物不是单纯前端问题，它会直接影响后端测试与编译行为。
