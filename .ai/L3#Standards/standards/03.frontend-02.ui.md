# 前端规范：UI

## 核心规则

- UI 变更需尊重当前仓库既有视觉语言与组件体系。
- 当前项目保留 React StrictMode 作为开发期检查手段。
- 如果第三方 UI 组件在 StrictMode 下持续触发已知弃用警告，并且升级库后仍未解决，可以针对单点场景改为自定义轻量实现。

## 实施要求

- 替换第三方组件时，应优先保持用户可见样式和交互一致，不做无关视觉重构。
- 当同类 StrictMode 问题在多个菜单场景重复出现时，应优先抽出项目级基础组件统一替代，而不是在每个业务文件里重复手写。
- 当前项目已建立 `AppDropdownMenu` 作为统一基础方案，用于替代在 StrictMode 下会触发 `findDOMNode` 警告的 `Semi UI Dropdown` 使用链路。
