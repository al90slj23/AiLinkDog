# 前端规范：技术栈与 UI

## 当前真实前端栈

- React 18
- Vite
- Semi Design UI
- Bun

## 明确事实

旧项目中与 Ant Design、禁用 Semi-UI 等相关规则不适用于当前仓库，不能照搬。

## 开发原则

- 前端依赖与脚本优先用 Bun
- UI 变更需尊重当前仓库既有视觉语言与组件体系
- 不在现有 Semi Design 项目里强行套入旧项目前端规则

## StrictMode 与第三方组件

- 当前项目保留 React StrictMode 作为开发期检查手段
- 如果某个第三方 UI 组件在 StrictMode 下持续触发已知弃用警告，并且升级库后仍未解决，可以针对单点场景改为自定义轻量实现
- 这种替换应优先保持用户可见样式和交互一致，不做无关视觉重构
- 当同类问题在多个菜单场景重复出现时，应优先抽出项目级基础组件统一替代，而不是在每个业务文件里重复手写
- 当前项目已建立 `AppDropdownMenu` 作为这一类场景的统一基础方案，用于替代在 StrictMode 下会触发 `findDOMNode` 警告的 `Semi UI Dropdown` 使用链路

## 明确禁止的错误继承

- 不能把旧项目“禁用 Semi-UI”的规则搬进当前仓库
- 不能把旧项目基于 Ant Design 的命名与组件约束直接套进当前项目
