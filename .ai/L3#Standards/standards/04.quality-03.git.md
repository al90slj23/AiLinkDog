# 质量规范：Git

## 基本原则

- 未经明确要求不自动提交 commit
- 不擅自 push
- 不擅自改写历史
- 只在必要范围内提交当前任务相关文件

## 当前仓库附加要求

- 提交前应有验证证据
- 如果仓库已有用户改动，不应随意回退
- `tmp/` 属于本地临时目录，不应纳入版本管理
- `L4#Changelog` 工作日志文件命名采用 `YYYY-MM-DD-HHMM-主题.md`
- 对大 diff / 大 prompt 的 AI 辅助链路，应优先做上下文收敛，避免触发本地参数长度或 provider 输入上限问题

## 安全披露

根据当前仓库 `.github/SECURITY.md`：

- 安全漏洞不应在公开 GitHub Issue 中披露
- 应通过 GitHub Security Advisories 或指定邮箱走私密披露流程

这类规则虽然不属于普通 commit 规范，但属于仓库层面的流程纪律，应被长期记忆。
