# 质量规范：协作

## 核心规则

- 当前规则主要来自 `.github/PULL_REQUEST_TEMPLATE.md`、`.github/ISSUE_TEMPLATE/bug_report.md` 与 `.github/ISSUE_TEMPLATE/feature_request.md`。
- 这些 GitHub 模板表达的是当前仓库真实的协作纪律，应进入长期主记忆。
- `Issue` 规则固定归入协作规范，而不是单列成独立文件。

## 实施要求

- 问题反馈与功能请求都要求遵循模板，不应随意删空模板结构。
- 提交前应先看文档、README 和现有问题列表，避免明显重复或低质量描述。
- `Bug fix` 不应被滥用来包装设计取舍、理解偏差或预期不一致。
- 重大新功能更适合先通过 Issue 沟通，再进入实现。
- 协作中倾向于要求提交者先自行做功课，再提出问题或提交变更。
