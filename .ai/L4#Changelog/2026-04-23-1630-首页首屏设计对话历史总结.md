### 5. 首屏布局多轮迭代

**两行三列尝试：**

**三列→两列重构：**
- 第一行改为两列：左主文案列（含指标）、右机器人主体
- 指标从独立中列并回左侧文案列

**背景层与内容层分离：**
- 机器人背景层从右列提升到第一行容器级别
- 使用 `position: absolute; left: 50%; width: 100vw; transform: translateX(-50%)`
- 背景铺满整行，内容层（文案/机器人）压在其上
- 供应商块限制在右半区（`left: 54%~66%`）

**最终确立的三行结构：**
- **第一行**：左文案 / 右机器人（背景铺满整行）
- **第二行**：左信息卡列 / 右代码框

## 关键判断与边界

1. **命名规范优先收敛**：在"第一行/第二行/第三行"、"主文案列/机器人主体/供应商块"的称呼统一后，后续多轮布局沟通效率大幅提升。

2. **背景层必须物理分离**：`100vw + translateX(-50%)` 是解决"内容容器有 padding 但背景要铺满"的最稳方案。

3. **不要给 Spline 额外包整机运动**：Spline 场景自己的交互（如头部跟随）和外部 CSS transform 会互相干扰，应只保留事件转发。

4. **供应商块要设 `pointer-events: none`**：中间视觉层如果吃事件，会阻断 Spline canvas 的鼠标交互。

5. **左列宽度不要人为压太窄**：为"让文案像左列"而把宽度压到 440px 以下会导致文案不合理换行，反而破坏体验。
