## 本次完成的核心事实

### 用户页能力推进

当前用户页已具备以下能力：

1. 独立进入邀请返利中心。
2. 首次锁定返利方案时显示确认弹窗。
3. 按账号级锁定返利方案，而不是按邀请链接锁定。
4. 锁定方案后创建邀请链接。
5. 查看总收益、可提现、待结算、邀请人数。
6. 查看佣金记录。
7. 查看邀请用户。
8. 发起提现申请。
9. 查看提现记录。

### 管理页能力推进

当前管理页已具备以下能力：

1. 查看邀请返利管理独立页面。
2. 查看顶部统计卡和方案运营概览。
3. 编辑全局返利配置并保存。
4. 新增返利方案。
5. 编辑返利方案。
6. 启用或停用返利方案。
7. 查看提现审核列表。
8. 审核通过或拒绝提现申请。

### 返利比例配置口径收敛

本轮把返利比例配置规则明确落到了代码与测试：

1. 最低利润率用于确定利润基数。
2. 利润分配比例用于确定实际返利池。
3. 一级返利占比与二级返利占比总和固定为 `100%`。
4. 前端编辑弹窗中，一级 / 二级返利占比会自动联动。
5. 后端保存方案时也会校验两者总和必须为 `100`。

### 充值后自动分佣接通

本轮最关键的工作是：

1. 把充值成功后的一级 / 二级返利生成链路接入后端。
2. 在真实样本 A -> B -> C 关系下验证：
   - C 充值 100 后，B 拿一级返利，A 拿二级返利。
3. 返利真实写入：
   - `referral_commissions`
4. 同步更新：
   - `users.aff_quota`
   - `users.aff_history`

实际验证结果：

1. 一级返利：`9`
2. 二级返利：`6`

这说明当前实现采用的是整数截断口径，而不是保留小数或四舍五入。

### 默认方案与统计口径收敛

本轮已修复：

1. 默认方案重复初始化问题。
2. 默认方案初始化误删管理员新增方案的问题。
3. 管理页顶部统计卡口径错误问题。
4. 管理页概览和返利方案职责重复的问题。
