# 2026-04-21 L3 standards 命名规则重构与单主题拆分

## 背景

在持续维护 `.ai/L3#Standards/standards/` 过程中，原有规范文件逐步暴露出两个问题：

1. 文件名长期混用“主题分组 + 复合短语”，例如 `json-and-dto`、`stack-and-ui`、`rule-sources-and-platform-files`。
2. 单篇文件经常同时承载多个长期规则对象，导致后续继续细分时，命名、索引和正文边界都变得不稳定。

同时，当前项目对规范正文有持续拆细的需求，说明旧命名方式已经不足以支撑长期演进。

## 本次确认的长期规则变化

本次工作正式确认并落地以下规则：

1. `L3#Standards/standards/` 正文文件统一采用可无限扩展的单词链命名。
2. 命名格式收敛为 `序号.单词-序号.单词...md`。
3. 每一层默认只使用一个英文单词，不再使用层内复合短语兜底。
4. 英文单词必须在阅读全文后提炼，而不是直接沿用旧文件名。
5. 如果正文存在并列主主题，应优先拆文或增层，而不是继续保留 `and` 式复合命名。

这意味着本次变化不是一次普通重命名，而是对长期规范组织方式的正式收敛。

## 本次结构调整结果

本次完成了以下结构性变化：

1. backend 规范拆分为 `json / dto / database / relay / channel / constants / config`。
2. frontend 规范拆分为 `stack / ui / i18n / build`。
3. quality 规范拆分为 `change / testing / git / collaboration / pr / page / evolution / parallel`。
4. tool 规范拆分为 `gogogo / rules / sources / platform / sync`。
5. AI memory 规范从 `10.ai-memory-*` 重构为 `10.ai-01.memory-*`，并拆出 `skills / platform / sync` 子层。

其中，原本混在旧 `10.ai-memory-02.usage.md` 中的“完成态同步”相关语义，被重新归位到：

- `10.ai-01.memory-03.skills-02.sync.md`

## 索引与入口同步

本次同步更新了以下入口：

1. `L2#Index/toc.md`
2. `L2#Index/conventions.md`
3. `L3#Standards/README.md`
4. `L3#Standards/standards/README.md`

更新后，`L2` 与 `L3` 的导航入口已经统一指向新命名体系。

## 协作流程层面的事实

本次整改最初曾在独立 worktree 中推进，但后续与主工作区同时出现了对 `.ai` 文档的并行修改，导致：

1. worktree 中的规范文档状态逐渐落后于主工作区。
2. 真正需要手工分析的重叠文件集中出现在 `L2` 索引与 `10.ai` 记忆规范上。
3. 最终需要把 worktree 中的整改结果再人工并回 `main`。

因此，本次工作形成了一个新的协作偏好事实：

- 对于容易与主工作区并行修改的文档体系，不再默认优先使用 worktree。
- worktree 更适合边界单一、改动大块且不易与主工作区发生同步冲突的场景。

这条内容目前属于协作事实与经验收敛，尚未单独上升为新的正式开发规范条文。
