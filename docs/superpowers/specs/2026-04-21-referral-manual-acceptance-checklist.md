# 邀请返利增强版人工验收清单

## 目的

本清单用于在当前项目中，对邀请返利增强版进行上线前人工回归。它只关注人工点击路径和预期结果，不替代高强度数据库验证计划。

验收页面：

1. 用户页：`/console/referralcenter`
2. 管理页：`/console/referralmanage`

## 验收前准备

1. 重启后端，确保最新迁移和统计逻辑已加载。
2. 重启前端，确保最新页面和弹窗逻辑已加载。
3. 使用管理员账号登录。
4. 确保数据库里已有至少一组样本用户与方案数据。

## 一、管理页验收

### 1. 页面可见性

- [ ] 左侧边栏能看到 `邀请返利管理`
- [ ] 打开 `/console/referralmanage` 页面成功

### 2. 概览页

- [ ] 顶部统计卡可见
- [ ] 统计卡包含：
  - `活跃邀请人`
  - `总邀请数`
  - `待处理提现`
  - `总返利支出`
- [ ] 下方方案运营概览可见
- [ ] 每个方案至少显示：
  - 方案名称
  - 方案类型
  - 选择人数
  - 一级返利累计
  - 二级返利累计
  - 总返利累计
  - 状态
- [ ] 概览页不出现：
  - `新增返利方案`
  - `编辑`
  - `启用/停用`

### 3. 返利方案页

- [ ] 页面中有 `新增返利方案`
- [ ] 每条方案有：
  - `编辑`
  - `启用/停用`
- [ ] 点击 `新增返利方案` 可弹出三栏编辑窗口
- [ ] 点击 `编辑` 可回显当前方案原始值
- [ ] 左栏有参数说明
- [ ] 中栏有方案名称、方案类型、方案说明和比例配置
- [ ] 右栏有预览和 100 元示例金额
- [ ] 修改一级 / 二级占比时，两者总和始终为 100%
- [ ] 保存成功后表格实时刷新

### 4. 返利配置页

- [ ] 能看到全局配置表单
- [ ] 至少可编辑：
  - 启用邀请返利
  - 默认链接有效期
  - 允许用户自定义链接有效期
  - 最低提现金额
  - 提现手续费比例
  - 固定手续费
  - 自动审核提现
  - 启用月度佣金上限
  - 月度佣金上限
- [ ] 点击保存后提示成功

### 5. 提现审核页

- [ ] 管理页能看到用户提交的提现申请
- [ ] `通过` 后状态更新
- [ ] `拒绝` 后状态更新并能看到管理员备注

## 二、用户页验收

### 1. 页面可见性

- [ ] 左侧边栏能看到 `邀请返利中心`
- [ ] 打开 `/console/referralcenter` 页面成功

### 2. 总览区

- [ ] 顶部说明文案完整可见
- [ ] 总览卡包含：
  - `总收益`
  - `可提现`
  - `待结算`
  - `邀请人数`
- [ ] 余额说明文案完整可见

### 3. 首次锁定方案

- [ ] 未锁定账号进入后能看到可选方案
- [ ] 点击某个方案时出现确认弹窗
- [ ] 弹窗包含：
  - `确认选择返利方案`
  - `重要提示`
  - 永久生效、无法更改提示
  - 当前方案名称、类型、一级返利、二级返利
- [ ] 点击 `确认选择` 后锁定成功
- [ ] 页面切换到已锁定方案状态

### 4. 邀请链接

- [ ] 已锁定账号能创建邀请链接
- [ ] 可填写渠道备注
- [ ] 可填写有效期
- [ ] 创建成功后列表出现新链接
- [ ] 列表至少显示：
  - 邀请链接
  - 渠道备注
  - 状态
  - 返利方案

### 5. 佣金记录

- [ ] 佣金记录页能显示数据
- [ ] 核对字段：
  - 用户名
  - 层级
  - 充值金额
  - 返利额度
  - 返利方案
  - 状态

### 6. 邀请用户

- [ ] 邀请用户页能显示数据
- [ ] 核对字段：
  - 用户名
  - 层级
  - 总充值
  - 总返利

### 7. 提现管理

- [ ] 可填写提现申请表单
- [ ] 余额不足时会被拦截
- [ ] 余额足够时可成功申请
- [ ] 提现记录列表可见

## 三、充值返利核心验收

- [ ] 准备 A -> B -> C 两级邀请关系
- [ ] C 完成一笔 100 元充值
- [ ] 验证 B 收到一级返利
- [ ] 验证 A 收到二级返利
- [ ] 管理页概览能看到方案累计返利变化
- [ ] 用户页佣金记录能看到新增流水

## 四、结果判断

只有当以下条件全部满足时，才可视为人工验收通过：

1. 用户页和管理页均可访问。
2. 用户首次锁定方案、创建链接、查看佣金、提现申请均可正常完成。
3. 管理员新增 / 编辑 / 启停方案与配置保存均可正常完成。
4. 充值后真实产生一级 / 二级返利。
5. 提现申请与审核在用户页和管理页状态一致。
6. 关键统计卡与方案运营概览显示正常。
