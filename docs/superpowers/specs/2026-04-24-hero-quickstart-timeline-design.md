# Hero Quick Start Layout Relocation & Design

## 背景与目标
根据用户的最新要求，将首屏原来左侧第二行（Secondary Left）的 `LandingHeroInfoCards` 移除（该组件原本展示：99.98% 30天可用率、128+ 模型数、< 80ms 开销以及 TRUST 供应商标志），转而把页面下方的 `LandingQuickStart` (快速接入的三步操作) 移动到这一区域。
同时，用户要求给这个三步接入块增加**设计感**，使之符合“快速、无门槛、步骤少”的直观印象，避免继续堆叠简单的框框（因为右边已经有很多框）。

## 原有结构分析
原来的 `LandingQuickStart` 是作为一个独立的 section 渲染的，拥有比较宽阔的横向分布（或者是单独的卡片堆叠）。
如果直接移入 `.ald-home-hero__secondary-left`（宽度约为 520px 的左侧网格细胞），其原本的设计可能不适用，或者会变得局促。

## 设计方案探索

### 方案 A：垂直卡片堆叠 (Vertical Cards)
将三步操作变成三个紧凑的小卡片，纵向排列。
**优点**：容易布局，适合窄列。
**缺点**：太像框框了，不符合用户“减少框框”以及“更有设计感”的要求。

### 方案 B：时间轴流线式 (Timeline / Flow)
使用一条发光的渐变线条串联三个步骤，去掉卡片的实体边框。步骤以图标或数字高亮，右侧跟随简短文字。
**优点**：极具流动性和引导性，打破了“框”的死板感。视觉上非常直观地展现了“过程”的轻盈。
**缺点**：需要精细调优 CSS 线条与节点。

### 方案 C：发光终端命令行样式 (Terminal Steps)
将三个步骤做成类似终端按顺序输出的三行语句。
**优点**：科技感强。
**缺点**：右侧已经是代码块了，左侧再搞类似终端的设计会造成视觉疲劳和重复。

## 推荐方案：方案 B (流动时间轴 Timeline Flow)
为了最大化呈现“无门槛”和“快速”，推荐剥离所有厚重的卡片背景和边框，使用一条贯穿上下的渐变线连接 3 个步骤节点。

**视觉细节：**
- **结构**：左侧一条发光的竖线（使用 `var(--ald-primary)` 到 `var(--ald-orange)` 的渐变）。在线上分布三个发光的圆点节点。
- **内容**：每个节点右侧对应一步操作，比如 `1. 获取 API Key` -> `2. 更改 Base URL` -> `3. 无缝调用`。
- **排版**：标题大且显眼，副标题字体略小颜色略暗 (`--ald-muted`)。取消任何外边框（无框设计）。
- **特效**：竖线可以使用 CSS animation 实现流光滑动的效果（类似于能量传输），节点上可以增加微微的 `box-shadow` 发光，代表激活状态。
- **字体**：标题使用粗体 Sans-serif，数字或英文部分可以使用 `IBM Plex Mono` 增加极客感。

这种布局极其轻盈，完美融入深色/浅色背景中，并且和右边严谨的代码框形成“左引导，右实践”的完美搭配。
