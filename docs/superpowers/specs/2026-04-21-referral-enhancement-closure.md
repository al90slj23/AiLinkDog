### 充值后自动分佣

当前已打通充值完成后的自动分佣链路：

1. 用户充值完成后，可按邀请关系生成一级返利。
2. 如果存在上上级邀请关系，可继续生成二级返利。
3. 佣金会真实写入 `referral_commissions`。
4. 同步更新到用户的：
   - `aff_quota`
   - `aff_history`

## 当前业务规则

### 方案选择

1. 用户首次创建邀请链接前必须先选择返利方案。
2. 返利方案一旦确认，按账号锁定，不可直接更改。
3. 该方案会应用到该账号下所有邀请链接。

### 返利比例配置

当前返利方案使用以下参数：

1. 最低利润率
2. 利润分配比例
3. 一级返利占比
4. 二级返利占比

当前规则：

1. 一级返利占比与二级返利占比总和固定为 `100%`。
2. 编辑弹窗中，修改其中一个值会自动联动另一个值。
3. 后端保存时也会校验两者总和必须等于 `100`。

### 返利金额口径

当前实现口径：

1. 利润基数 = `充值金额 * 最低利润率`
2. 返利池 = `利润基数 * 利润分配比例`
3. 一级返利 = `返利池 * 一级返利占比`
4. 二级返利 = `返利池 * 二级返利占比`

当前代码在写入佣金额度时采用整数截断：

1. `9.6 -> 9`
2. `6.4 -> 6`

这属于当前实现口径，后续如果需要改为四舍五入或更细粒度货币单位，需要单独调整。

### 仍然与旧字段协同工作的部分

当前返利增强版仍与旧用户字段并存：

1. `users.aff_quota`
2. `users.aff_history`
3. `users.aff_count`
4. `users.inviter_id`

其中：

1. `aff_quota / aff_history` 当前仍作为用户返利余额和历史总额的直接承载字段之一。
2. `inviter_id` 仍参与邀请关系和邀请用户统计。

这意味着当前系统仍处在“新 referral 域与旧 user 邀请字段协同”的阶段，而不是完全切断旧字段依赖。
